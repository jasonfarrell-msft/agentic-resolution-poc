## Schema

Single index: `tickets-index` with 14 fields:
- **Keyword fields:** id, number, shortDescription, description, category, priority, state, assignedTo, caller, resolutionNotes
- **Vector field:** contentVector (1536d, text-embedding-3-small)
- **Metadata:** createdAt, updatedAt (for filtering/sorting)

## Hybrid Search Strategy

- **Vector embedding:** Concatenate `shortDescription + " " + description + " " + category`
- **Query mode:** BM25 (full-text) + vector similarity (cosine, HNSW)
- **Reranking:** Semantic ranker (`ticket-semantic` config: title=shortDescription, content=[description, resolutionNotes], keywords=[category, number])
